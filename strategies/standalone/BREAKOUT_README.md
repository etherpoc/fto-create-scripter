## ロジック

- **確定 H1 足ベース**(チャート足に非依存。`iX(_Symbol, PERIOD_H1, shift)` で取得)。
- **エントリー(flat時)**: 前 entry_n 本の高値を当足高値がブレイク かつ close > SMA(sma_n)
  → 成行ロング。SL = fill − sl_atr×ATR。**long-only**。
- **エグジット**: 前 exit_n 本安値のトレール割れ、または SL ヒット。
- **サイズ**: 口座 risk% を SL距離で逆算。**`OrderCalcProfit` で口座通貨建ての実損を正確計算**
  (SYMBOL_TRADE_TICK_VALUE はゴールド×非USD口座で誤値を返すため使わない)。

> ※ バックテストは Donchian level 約定、本EAは確定足 close 成行約定なので**実機はわずかに保守的**。

## セットアップ

1. **コンパイル**: MetaEditor で `breakout_h1.mq5` を開き F7(0 errors/0 warnings)。
2. **5ペアを各 H1 チャートに 1 つずつ**適用: XAUUSD / USDJPY / EURJPY / AUDJPY / GBPJPY。
   - チャート足は実は任意(EAが H1 を明示取得)だが H1 推奨。
3. **パラメータは既定のまま**(en30/ex25/SL3/SMA150/risk0.5%/long-only)。
4. **起動後の確認(最重要)**: Journal の `[bo_h1] ENTRY long ... risk<ccy>=X 実損≈Y ...` で
   **`実損≈` が `risk<ccy>`(=残高の0.5%)とほぼ一致**しているか目視
   (ここがズレるとサイジング異常。time_of_day 全損の教訓)。

### ★ テスター注意: 証拠金を実運用サイズに
**口座が小さすぎる(例 ¥100,000 ≈ $670)と、最小ロットが risk% を超えて発注拒否(0トレード)になる。**
**Deposit は ¥10,000,000(≈$67k)や $100,000 など実運用サイズに**。Python の数値は口座の % なので
十分大きければ正しくスケールする。

### CSV ログ

`InpCsvLog=true` で `MQL5/Files`(テスターは Tester 配下)に entry/exit を保存。
