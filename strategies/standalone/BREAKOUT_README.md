# MT5 EA — H1 Donchian ブレイクアウト (`breakout_h1.mq5`)

研究B(6-8%/月を狙う別ロジック探索)で見つけた **H1 long-only ブレイクアウト・バスケット**の
MT5 実装。MT5 実機(Axiory)+実コストで検証済み。プロジェクト全体の最高到達点。

- 本体: [`breakout_h1.mq5`](./breakout_h1.mq5)

### 期待値(実Axiory 11年・overlay入り・DD10%スケール)

| basket | OOS(2015-20) | IS(2021-26) | 効率(sumR/DD) |
|---|---|---|---|
| BO5(旧) | +0.84%/月 | +2.01%/月 | 18.9 |
| **BO7(採用)** | **+1.24%/月** | **+2.32%/月** | **23.7** |

- **採用 = 7ペア・risk ~0.3%/pair で総DD<10%**(0.5%/pairならリターンもDDも比例増)。
- ★ **レジーム依存は実データでも明確**: IS(円安/金高期)は OOS の約2倍稼ぐ。**強トレンド期に大・平時に小**が正しい期待値
  (USDJPY単体は OOS ほぼ平ら)。「毎月コンスタント」ではない。
- ★ **稼ぐペアはレジームで移動する**: 2015-20は USD系・EURGBP も稼いだが 2021-26 で死亡、JPY/金に集中。
  → ペア選択を固定し過ぎない。現robust-7も将来ドリフトしうる前提で。

### ★ ショート・スリーブ (`InpShortOnly`) — risk調整後リターン約2倍 (2026-06-14, 実験的)

long-only は上昇相場が要る単一レジームベット。**robust-7 のショート side は long と月次相関 -0.18(負=ヘッジ)・
両期間 net+**。long + **0.4×short** スリーブで DD10%換算 **OOS +1.24→+2.64 / IS +2.32→+3.72%/月**、
効率 23.7→42.7(**約2倍**, `tools/axiory_longshort.py`)。α=0.2-0.5でノブ非依存=過剰適合でない。

**運用(ヘッジ口座前提)**: 既存7チャートの long EA(magic=220612, risk0.3-0.5%)に加え、
**同じ7チャートに short 専用 EA を重ねる**: `InpShortOnly=true` / `InpMagic=別値`(例220613) /
`InpRiskPct = longの0.4倍`(例0.12-0.2%)。init ログに `dir=SHORT-only` 確認。
- ⚠ **ネッティング口座は long/short が相殺**するので不可(ヘッジ口座のみ)。
- ⚠ **まだPythonバックテスト・MT5実機未検証。** ショートはギャップ/スクイーズの裾リスクあり → **実機で要検証**。
- 詳細・検証は `docs/PROGRESS.md`「H章」。

## ロジック

- **確定 H1 足ベース**(チャート足に非依存。`iX(_Symbol, PERIOD_H1, shift)` で取得)。
- **エントリー(flat時)**: 前 entry_n 本の高値を当足高値がブレイク かつ close > SMA(sma_n)
  → 成行ロング。SL = fill − sl_atr×ATR。**long-only**。
- **エグジット**: 前 exit_n 本安値のトレール割れ、または SL ヒット。
- **サイズ**: 口座 risk% を SL距離で逆算。**`OrderCalcProfit` で口座通貨建ての実損を正確計算**
  (SYMBOL_TRADE_TICK_VALUE はゴールド×非USD口座で誤値を返すため使わない)。

> ※ バックテストは Donchian level 約定、本EAは確定足 close 成行約定なので**実機はわずかに保守的**。

## セットアップ

1. **コンパイル**: MetaEditor で `breakout_h1.mq5` を開き F7(0 errors/0 warnings)。
2. **7ペアを各 H1 チャートに 1 つずつ**適用: XAUUSD / USDJPY / EURJPY / AUDJPY / GBPJPY / CHFJPY / NZDJPY。
   - チャート足は実は任意(EAが H1 を明示取得)だが H1 推奨。
3. **パラメータは既定のまま**(en30/ex25/SL3/SMA150/risk0.5%/long-only)。
4. **起動後の確認(最重要)**: Journal の `[bo_h1] ENTRY long ... risk<ccy>=X 実損≈Y ...` で
   **`実損≈` が `risk<ccy>`(=残高の0.5%)とほぼ一致**しているか目視
   (ここがズレるとサイジング異常。time_of_day 全損の教訓)。

### ★ テスター注意: 証拠金を実運用サイズに
**口座が小さすぎる(例 ¥100,000 ≈ $670)と、最小ロットが risk% を超えて発注拒否(0トレード)になる。**
**Deposit は ¥10,000,000(≈$67k)や $100,000 など実運用サイズに**。Python の数値は口座の % なので
十分大きければ正しくスケールする。

### Fintokei: 同時保有リスク上限 (`InpMaxTotalRiskPct`, 既定3.0)

プロップ(Fintokei)の「同時に持てるリスク 3% まで」ルール用。新規エントリー時に
**口座全体の保有中ポジションのSL基準リスク合計 + 新規リスク が上限を超えるならスキップ**する。
- 対象は**全シンボル・全 magic・手動分も含む口座全体**(各ポジションの open→SL 損失を `OrderCalcProfit` で算定)。
- 7ペア各0.3-0.5% = 最大2.1-3.5%。手動建てや他EAと併用時の保険。0=off。
- スキップ時は Journal に `[bo_h1] SKIP(Fintokei) ... 保有リスクX% + 新規Y% > 上限3%`。

### ★ エクイティカーブ・デリスク overlay (`InpOverlay`, 既定 true)

研究E章(2026-06-14)で検証した DD 削減ロジック。**口座エクイティが直近 `InpOvDays`(既定60)日の
移動平均を割ったら、新規エントリーのロットを `InpOvMult`(既定0.5)倍**にする。

- **狙い**: トレンド戦略のDDは凍れ(chop)・レジーム減衰期にクラスタする。そこで縮小すると
  DDだけ非対称に減る。検証で **BO basket DD 32→20%(全年一貫)・シャッフル対照で「本物(連続DD捉え)」と確認**。
- **口座エクイティ基準**=全EAインスタンスが同じ口座エクイティから計算=**バスケット/口座レベルで機能**
  (5枚運用でも口座全体のDDで一斉に縮小)。**スケール不変**なので絶対閾値のチューニング不要。
- 起動直後は `InpOvDays/3` 日ぶんのサンプルが貯まるまで通常サイズ(overlay非作動)。
- entry ログに `overlay=x0.5(eq=... MA=...)` が出る。**`eq < MA` のとき x0.5 になっているか目視確認**。
- `InpOverlay=false` で従来挙動(常時フルサイズ)に戻せる。

#### ★ 段階的な運用(過剰適合を踏まない手順)
1. **まず overlay ON のまま `InpRiskPct=0.5` を維持** → リターンほぼ不変で **DD が約9.4%→6%級に下がる**(安全側の改善)。
2. **数か月フォワードで「DDが実際に下がる」ことを実機確認**してから、空いたDD枠ぶん **risk% を上げて再レバ**
   (検証上は 0.5→0.7-0.8%で月利が比例増)。**順序厳守**。検証なしの再レバ+overlay同時投入はしない。
- 注: 二層overlay(月+8%)は研究で**過剰適合**と判定し**不採用**。EAは単層(口座エクイティMA)のみ。

## ⚠️ リスクと位置づけ(誇張なし)

- **最大の懸念=レジーム依存**。viable は実質「金ロング + 円ショート群」で、2021-26 の金高/円安
  トレンドに乗っている。**マクロ反転(円高・金安)で同時逆行**(2026の鈍化が予兆かも)。
- **6-8%/月は安全DDでは未達**(1%/pairで月+5.2%だがDD18.8%)。**0.5%で月+2.62%/DD9.4%が現実解**。
