# Strategy Spec: zigzag_line_break

ZigZag + ライン分析 + MTF 環境認識を組み合わせたダウ理論ベースの売買戦略。

---

## 概要
- 一行サマリ: ライン付近で Z1 (大きい ZigZag) がダウ転換した直後に、
  Z2 (細かい ZigZag) の順スイングを抜けた瞬間にエントリ。
- 着想元: ユーザ提示のロジック (Untitled-2)。

## 対象
- 通貨ペア: 任意。サンプルは架空 5 桁ペア (pip_size = 0.0001)。
- 主時間足: **M15**。
- 上位足 (MTF):
  - 使う: **H1** と **H4**。
  - 用途:
    - 上位足の Z1 ピボット価格を「強い水平ライン」として扱う (環境認識)。
    - ライン付近で順方向のエントリをフィルタする (壁前禁止)。
  - 参照ルール: 上位足の確定足のみを下位足インデックスに展開して使う
    （未確定上位足の参照禁止）。

## インジケータ設定 (ZigZag)
- **Z1 (大波 / トレンド判定用)**: depth=25, deviation=5 pips
- **Z2 (細波 / エントリトリガ用)**: depth=5, deviation=4 pips
- 上位足 (H1, H4): Z1 のみ。M15 と同じ depth=25, deviation=5 pips を適用。
- 実装は MT4 風 (simplified)。詳細は `ZigZagTracker` の docstring 参照。
  - depth = 中心ピボット候補の両側に必要な「より低い高値 / より高い安値」連続本数。
  - deviation = 直前の逆方向ピボットからの最小スイング幅 (価格単位、pip 換算可)。
  - ピボットは右に depth 本進んで初めて確定 (= ルックアヘッドなし)。

## 環境認識 (ラインの作り方)
- ライン = Z1 のピボット価格 (M15 / H1 / H4 すべて)。
- **「ライン付近」の判定**: `|price - line| ≤ ATR(14) × line_atr_k`
  既定 `line_atr_k = 0.5` (パラメータ化)。

## トレンド (Dow 理論) 定義
- 「Dow トレンド」は Z1 の直近 2 ピボット間で判定する状態量。
  - 直近ピボットが高値で、その前の高値より高い → 上昇継続。
  - 直近ピボットが安値で、その前の安値より低い → 下降継続。
  - 上昇中に **「価格 (close) が直近 Z1 安値を下抜け」** → ダウ転換 (下降へ)。
  - 下降中に **「価格 (close) が直近 Z1 高値を上抜け」** → ダウ転換 (上昇へ)。
- 転換認識は **確定足の close** で判定 (wick では認識しない)。

## ロングエントリー条件
すべて AND:
1. **ライン付近**: 直近 close が M15 / H1 / H4 のいずれかの Z1 ピボット価格から
   ATR×line_atr_k 以内。
2. **Z1 ダウ転換 (下降→上昇)** がついさっき発生した (直近 lookback_bars 本以内、既定 30 本)。
3. **Z2 順スイング超え**: 転換以降に確認された直近 Z2 高値を、
   現在の確定足 close が上抜けた。
4. **上位足の壁前ではない**: 現在 close からエントリ方向 (上) に
   ATR × wall_atr_k 以内に H1 / H4 の Z1 高値 (=抵抗) が存在しない。
   既定 `wall_atr_k = 1.0`。
5. 転換からエントリトリガまでの間に **逆方向の Z1 転換が発生していない**。

## ショートエントリー条件
ロングと対称:
1. ライン付近 (同条件)。
2. Z1 ダウ転換 (上昇→下降) が直近 lookback_bars 以内。
3. Z2 順スイング (Z2 安値) を close で下抜け。
4. 下に H1 / H4 の Z1 安値 (=支持) が ATR × wall_atr_k 以内にない。
5. 転換以降に逆方向 Z1 転換なし。

## エグジット
- **TP**: エントリ方向の「次の意識されるライン」価格。
  - 探索優先順位: H4 Z1 > H1 Z1 > M15 Z1。
  - ロング: エントリ価格より上、かつ最も近いもの。
  - ショート: エントリ価格より下、かつ最も近いもの。
  - 見つからない / 距離が極端に小さい (< SL 距離 × min_rr) なら、
    SL 距離 × tp_rr の固定 RR にフォールバック。既定 `tp_rr = 2.0`、`min_rr = 1.0`。
- **SL**:
  - ロング: 転換のきっかけになった直近 Z1 安値 − ATR × sl_buffer_k。
  - ショート: 同じく直近 Z1 高値 + ATR × sl_buffer_k。
  - 既定 `sl_buffer_k = 0.5`。
- **時間ベース決済**: なし。
- **反対 Z1 ダウ転換でクローズ**: あり (ドテンはしない、クローズのみ)。

## リスク / 数量
- ロット = `account_balance × risk_pct / (SL 距離_pips × pip_value)`。
  既定 `risk_pct = 0.01` (1%)。
- 同時保有: 1。
- ドテン: しない (反対 Z1 転換はクローズのみ。次のシグナルで改めて入る)。
- 同方向の追加発注: 無視。

## 最適化パラメータ
| パラメータ名 | 既定値 | 探索範囲 | 備考 |
|---|---|---|---|
| z1_depth | 25 | 15..40 | Z1 ZigZag depth |
| z1_dev_pips | 5.0 | 2..15 | Z1 deviation (pips) |
| z2_depth | 5 | 3..15 | Z2 depth |
| z2_dev_pips | 4.0 | 1..10 | Z2 deviation (pips) |
| atr_period | 14 | 7..30 | ATR 期間 |
| line_atr_k | 0.5 | 0.2..1.5 | ライン付近判定の許容 |
| wall_atr_k | 1.0 | 0.5..2.0 | 上位足壁前の許容 |
| sl_buffer_k | 0.5 | 0.0..2.0 | SL の Z1 ピボット外側バッファ |
| tp_rr | 2.0 | 1.0..5.0 | TP フォールバックの RR |
| min_rr | 1.0 | 0.5..2.0 | ライン TP 採用の最小 RR |
| lookback_bars | 30 | 10..100 | 転換から Z2 トリガまでの最大待ち本数 |
| risk_pct | 0.01 | 0.001..0.03 | 1 トレードあたり口座リスク |

## 検証スコープ
- バックテスト期間: 約 1 年分の M15 サンプル CSV (自動生成、seed 固定)。
- 想定スプレッド / 手数料: ゼロ (足場検証目的)。
- 通貨換算: pip_size = 0.0001, pip_value = 10.0。
- 初期残高: 10,000。
- 既知のリスク: サンプルデータは乱数生成のため、勝ち負け数値自体に意味はない。
  「ロジックがエラーなく動き、適切にトレード回数が出ること」を主目的とする。

## FTO へ持って行くとき特に注意したい点
- FTO の配列 index 0 は最新。`src/core/` は古→新前提なので、
  `Close(i)` を `i=0..n-1` で集めたら **reverse して** 渡す。
- `OnTick()` はティック発火なので、`Time(0)` 保存比較で確定足判定を入れる。
- ポジション照会用関数は無いので、`getActiveOrderCount` → `selectOrder` →
  `getOrderType` を `magicNumber` フィルタ付きで回す。
- SL / TP は **価格指定** (本戦略のロジックと一致)。
- 上位足の参照は **確定足のみ**。`iHigh` / `iLow` の timeframe 引数の単位は
  ドキュメントの記載に揺れがあるため、FTO 側で **実値を貼って確認する** こと。
  本ロジックでは M15 → H1, M15 → H4 のみ使う。
